<async, static> exports.dataset( [settings])

gets a dataset from a datastore via given settings

The original settings given are not changed (they are cloned).
If the settings do not contain a dataset key, a unique key is generated.
If the dataset does not exist in the datastore, an empty dataset is returned. This dataset is not newly created in the datastore and is only returned locally.
Instead of the settings, a dataset can be given directly. This dataset is then returned as result.
If the dataset key is specified in the settings directly as the dataset, this dataset is returned as the result.
An instance for user authentication that can be reached from the datastore is automatically detected.

Parameters:
Name Type Argument Default Description
settings Object <optional>
 {}

contains the required data to determine the dataset (or is directly the dataset)

Properties
Name Type Argument Description
store Object

the datastore that contains the dataset
key * <optional>

the key of the dataset in the datastore (or initial dataset)
login boolean <optional>

The user must log in if he is not already logged in to receive the dataset (only if an instance for user authentication could be determined automatically).
user boolean <optional>

The dataset key given in the settings is expanded to a user-specific key: [ dataset_key, user_key ] (only if user is detected and logged in)
permissions Object <optional>

If the dataset does not exist, the empty dataset then returned will contain these permission settings.
convert function <optional>

With this function, the data contained in the result dataset can be adjusted.

Examples
// load dataset by datastore and key
const dataset = await dataset( { store: datastore, key: dataset_key } );
console.log( dataset );  // => { key: dataset_key, ... }
// passed dataset is directly returned as the result
const dataset = await dataset( { key: dataset_key, ... } );
console.log( dataset );  // => { key: dataset_key, ... }
// passed dataset as key is directly returned as the result
const dataset = await dataset( { store: datastore, key: { key: dataset_key, ... } } );
console.log( dataset );  // => { key: dataset_key, ... }
// load of an user-specific dataset
const dataset = await dataset( {
  store: datastore,
  key: dataset_key,
  login: true,       // user must log in if not already logged in
  user: true         // dataset key and user key will be combined
} );
console.log( dataset );  // => { key: [ dataset_key, user_key ], ... }
// get new empty dataset with generated key that contains only given permission settings
const dataset = await dataset( {
  store: datastore,
  permissions: { creator: 'john', realm: 'guest', access: 'creator' }
} );
console.log( dataset );  // => { key: generated_key, _: { creator: 'john', realm: 'guest', access: 'creator' } }
// adaptation of the data structure of the loaded dataset using a convert function
const dataset = await dataset( {
  store: datastore,
  key: dataset_key,
  convert: dataset => { dataset.lang = dataset.lang.toUpperCase(); return dataset; }
} );
console.log( dataset );  // => { key: dataset_key, lang: 'EN', ... }

<async, static> exports.onFinish(settings [, results])

allows a declarative way to perform common finish actions

The original parameters given are not changed (they are cloned).
If an instance is passed for settings, the finish actions defined via the instance.onfinish property are used.
If a function is passed for settings, the function is called with the result data.
If an instance is passed for settings, the result data are automatically determined via instance.getValue(). To do this, the instance must have a getValue method.
If an instance is passed for settings, the nearest instance for user authentication is automatically determined.

Parameters:
Name Type Argument Description
settings Object | function

declarative settings for usual finish actions (or 'onfinish' callback or finished instance)
results Object <optional>

result data of the finished instance
settings.confirm string <optional>

show confirm box (no finish actions will be performed if user chooses abort)
settings.condition function <optional>

no finish actions will be performed if this function returns a falsy value (result data and possibly the instance is passed as parameters)
settings.login boolean <optional>

user will be logged in if not already logged in (only works if an instance for user authentication could be determined)
settings.convert function <optional>

for dynamic adjustment of the results data (result data is passed as parameter, must return adjusted result data)
settings.log boolean <optional>

log result data in the developer console of the web browser
settings.clear Object <optional>

clear webpage area of the finished instance
settings.store Object | boolean <optional>

use this to store the result data in a data store (use boolean true to apply the settings of instance.data")

Properties
Name Type Argument Description
settings Object

datastore accessor configuration for a datastore (result data will be set in this datastore)

If the dataset does not exist, the dataset then will created with these permission settings.
settings.alert string <optional>

show alert message
settings.restart boolean <optional>

restart finished instance

render other content (ccmjs-based app or HTML content, as default the content is rendered in the root element of the instance)

additional finish callback which will be called after the other finish actions (result data and possibly the instance is passed as parameter)
