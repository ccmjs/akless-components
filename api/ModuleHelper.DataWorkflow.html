    static exports.dataset(settingsopt) → {Promise.<Object>}
    helper.mjs, line 412

    gets a dataset from a datastore via given settings

    The original settings given are not changed (they are cloned).
    If the settings do not contain a dataset key, a unique key is generated.
    If the dataset does not exist in the datastore, an empty dataset is returned. This dataset is not newly created in the datastore and is only returned locally.
    Instead of the settings, a dataset can be given directly. This dataset is then returned as result.
    If the dataset key is specified directly in the settings as the dataset, this dataset is returned as the result.
    A user instance that can be reached from the datastore is automatically detected.
    Parameters:
    Name Type Attributes Default Description
    settings Object <optional>
    		 {} contains the required data to determine the dataset (or is directly the dataset)
    Properties
    Name Type Attributes Description
    store Object the datastore that contains the dataset
    key * <optional>
    		 the key of the dataset in the datastore (or initial dataset)
    login boolean <optional>
    		 The user must log in if he is not already logged in to receive the dataset (only if a user instance could be determined automatically).
    user boolean <optional>
    		 The dataset key given in the settings is expanded to a user-specific key: [ dataset_key, user_key ] (only if user is detected and logged in)
    permissions Object <optional>
    		 If the dataset does not exist, the empty dataset then returned will contain these permission settings.
    convert function <optional>
    		 With this function, the data contained in the result dataset can be adjusted.
    Throws:
    if user must log in and login is canceled
    Type
    Error
    Returns:
    Promise.<Object>
    Examples
    await dataset( { store: datastore, key: dataset_key } )  // => { key: dataset_key, ... }
    await dataset( { key: dataset_key, ... } )  // => { key: dataset_key, ... }
    await dataset( { store: datastore, key: { key: dataset_key, ... } } )  // => { key: dataset_key, ... }
    await dataset( {
  store: datastore,
  key: dataset_key,
  login: true,
  user: true
} )  // => { key: [ dataset_key, user_key ], ... }
    await dataset( {
  store: datastore,
  permissions: { creator: 'john', realm: 'guest', access: 'creator' }
} )  // => { key: generated_key, _: { creator: 'john', realm: 'guest', access: 'creator' } }
    await dataset( {
  store: datastore,
  key: dataset_key,
  convert: dataset => { dataset.lang = dataset.lang.toUpperCase(); return dataset; }
} )  // => { key: dataset_key, lang: 'EN', ... }

    static exports.onFinish(settings, resultsopt) → {Promise.<void>}
    helper.mjs, line 544

    allows a declarative way to perform usual finish actions

    The original parameters given are not changed (they are cloned).
    If a ccm instance is passed for settings, the finish actions defined via the instance.onfinish property are used.
    If a function is passed for settings, the function is called with the result data.
    If a ccm instance is passed for settings, the result data are automatically determined via instance.getValue(). To do this, the instance must have a getValue method.
    If a ccm instance is passed for settings, the nearest user instance in the ccm context of the instance is automatically determined.
    Parameters:
    Name Type Attributes Description
    settings Object | function declarative settings for usual finish actions (or 'onfinish' callback or finished ccm instance)
    results Object <optional>
    		 result data of the finished ccm instance
    settings.confirm string <optional>
    		 show confirm box (no finish actions will be performed if user chooses abort)
    settings.condition function <optional>
    		 no finish actions will be performed if this function returns a falsy value (result data and possibly the ccm instance is passed as parameters)
    settings.login boolean <optional>
    		 user will be logged in if not already logged in (only works if a user instance could be determined)
    settings.convert function <optional>
    		 for dynamic adjustment of the results data (result data is passed as parameter, must return adjusted result data)
    settings.log boolean <optional>
    		 log result data in the developer console of the web browser
    settings.clear Object <optional>
    		 clear website area of the finished ccm instance
    settings.store Object | boolean <optional>
    		 use this to store the result data in a data store (use boolean true to apply the settings of instance.data")
    Properties
    Name Type Attributes Description
    settings Object settings for a ccm datastore (result data will be set in this datastore)
    key * <optional>
    		 dataset key for result data in the datastore (default is generated key)
    user boolean <optional>
    		 The dataset key is expanded to an user-specific key: [ dataset_key, user_key ] (only if user is detected and logged in)
    unique boolean <optional>
    		 The dataset key is expanded with an unique hash: [ dataset_key, user_key, unique_hash ]
    permissions Object <optional>
    		 If the dataset does not exist, the dataset then will created with these permission settings.
    settings.alert string <optional>
    		 show alert message
    settings.restart boolean <optional>
    		 restart finished ccm instance
    settings.render Object | * <optional>
    		 render other content (ccm-based app or HTML content, as default the content is rendered in the root element of the instance)
    settings.callback callback <optional>
    		 additional finish callback which will be called after the other finish actions (result data and possibly the ccm instance is passed as parameter)
    Returns:
    Promise.<void>
    Examples
    instance.onfinish = {
  confirm: 'Are you sure?',
  condition: ( results, instance ) => true,
  login: true,
  convert: json => json,
  log: true,
  clear: true,
  store: {
    settings: { name: 'store_name', url: 'path/to/server/interface.php' },
    key: 'dataset_key',
    user: true,
    unique: true,
    permissions: {
      creator: 'john',
      group: {
        john: true,
        jane: true
      },
      access: {
        get: 'all',
        set: 'group',
        del: 'creator'
      }
    }
  },
  alert: 'Finished!',
  restart: true,
  render: {
    component: 'component_url',
    config: {...}
  },
  callback: ( results, instance ) => console.log( results, instance )
};
onFinish( instance );
    instance.data = {
  store: { name: 'store_name', url: 'path/to/server/interface.php' }
  key: 'dataset_key',

  component: 'component_url',
  config: {
    root: document.body,
    ...
  }
} } );
    instance.onfinish = { render: 'Hello <b>World</b>!' };
onFinish( instance );

onFinish( instance );
